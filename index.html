Petful

펫 인플루언서 & 마케팅 플랫폼

개발 기간: 8주

팀 구성: 5명

담당 역할: API Gateway, User Service, JWT 토큰 구축, Pet Service, FTP 파일 서버 연동 (풀스택)

MSA

주요 기술

Spring Boot Spring Cloud Spring Security Next.js MySQL Redis OpenFeign Kubernetes

담당 개발 영역

  • API Gateway: JWT → X-User-No 헤더 변환으로 성능 최적화된 인증 시스템
  • User Service: JWT 토큰 구축, 이메일 인증, 비밀번호 재설정
  • Pet Service: 펫 포트폴리오, 히스토리 관리, 인스타그램 연동
  • FTP 파일 서버: 이미지 업로드/다운로드, 파일 관리 시스템
  • 서비스 간 통신: OpenFeign을 통한 User/Pet 서비스 연동
  • 프론트엔드: React 기반 사용자 인터페이스, API 연동, 상태 관리

트러블슈팅

!
JWT 토큰 인증 실패

Config Service와 User Service 간 JWT 시크릿 키 불일치로 401 에러 발생

해결: 환경변수 통일 및 Gateway에서 X-User-No 헤더 변환으로 성능 최적화

!
서비스 간 통신 타임아웃

OpenFeign 통신에서 ReadTimeoutException 및 연쇄 장애 발생

해결: Circuit Breaker 패턴 적용, 타임아웃 설정 조정, @Retryable로 재시도 로직 구현

!
파일 업로드 & 데이터 동기화

펫 이미지 업로드 실패 및 SNS 연동 데이터 동기화 문제

해결: FTP 서버 연결 상태 점검, @Transactional 처리, 외부 API 재시도 로직으로 안정성 확보

!
성능 최적화 & 모니터링

N+1 쿼리 문제 및 서비스 상태 모니터링 어려움

해결: @Query로 LEFT JOIN FETCH 사용, @Cacheable로 Redis 캐싱, CustomHealthIndicator로 헬스체크 구현

팀 구성: 5명

담당 역할: Review Service, FTP 파일 서버 연동 (풀스택)

Commerce

주요 기술

Spring Boot JPA Entity MySQL Next.js TypeScript Tailwind CSS Axios JWT Authentication

담당 개발 영역

Review Service: 리뷰 CRUD, JWT 인증, 권한 관리 (본인 리뷰만 수정/삭제)
FTP 파일 서버: 리뷰 이미지 업로드/다운로드, 파일 관리 시스템

API 설계: RESTful API 설계, Response DTO 통일, CORS 설정
프론트엔드: Next.js 기반 리뷰 UI/UX, API 연동, 상태 관리

트러블슈팅

!
리뷰 조회 성능 저하 (N+1 문제)

상품 상세 페이지에서 리뷰 목록 조회 시 N+1 쿼리로 인한 성능 저하

해결: @EntityGraph와 fetch join을 활용하여 한 번의 쿼리로 리뷰와 사용자 정보를 함께 조회하도록 최적화

!
API 응답 구조 불일치 & CORS 문제

프론트엔드와 백엔드 간 API 응답 구조 불일치 및 CORS 정책 오류

해결: 통일된 Response DTO 설계 및 @CrossOrigin 설정으로 API 일관성 확보 및 CORS 문제 해결

!
리뷰 권한 관리 & 파일 업로드 보안

본인 리뷰만 수정/삭제 가능하도록 권한 관리 및 파일 업로드 보안 강화 필요

해결: JWT 토큰 기반 사용자 인증 및 파일 타입/크기 검증 로직 구현으로 보안성 향상

!
상태 관리 복잡성 & UX 개선

장바구니, 쿠폰 등 복잡한 상태 관리와 사용자 경험 개선 필요

해결: Context API 활용한 전역 상태 관리 및 로딩 상태, 에러 처리 개선으로 사용자 경험 향상
